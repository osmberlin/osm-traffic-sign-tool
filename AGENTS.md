# AGENTS.md

## Cursor Cloud specific instructions

This is a **static client-side React SPA monorepo** — no databases, Docker, or backend services required.

### Services

| Service | Command | Port |
|---------|---------|------|
| Vite dev server (main app) | `pnpm run dev` (from root, runs all watchers via turbo) | 5173 |

Running `pnpm run dev` from the repo root starts turborepo in parallel mode: the Vite dev server for the app plus TypeScript watchers for all packages.

### Key commands

- **Install deps**: `pnpm install`
- **Build all**: `pnpm run build`
- **Dev (all watchers)**: `pnpm run dev`
- **Lint**: `pnpm run lint` (runs oxlint across all packages)
- **Format check**: `pnpm run format:check` (runs oxfmt; note `routeTree.gen.ts` is auto-generated and may fail format check)
- **Format fix**: `pnpm run format`
- **Test**: `pnpm run test` (vitest across all packages)
- **Pre-push check**: `pnpm turbo check` (what the husky pre-push hook runs)

### Gotchas

- **Node version**: pnpm manages the Node version via `.npmrc` (`use-node-version=22.21.1`). The system Node just needs to be >=22.
- **Bun**: Required for helper scripts (`script-new-data.ts`, release CLI, data fetching in internal packages). Not needed for standard dev/test/lint/build.
- **esbuild warning**: On fresh `pnpm install`, you may see a warning about ignored esbuild build scripts. The `pnpm.onlyBuiltDependencies` in root `package.json` only lists `sharp`. Despite the warning, Vite and Vitest work correctly because esbuild provides a WASM fallback.
- **Package build order**: `pnpm run build` uses turborepo's `dependsOn: ["^build"]` to correctly order the build. The converter package copies SVGs from `packages/internal_svgs/src/data-svgs/` (committed to repo).
- **`routeTree.gen.ts`**: Auto-generated by TanStack Router plugin during `vite dev`. Do not manually edit; format check failures on this file are expected.
